課題

簡単に識別できる。でも「場所」がわからない。

Teachable Machineは画像分類をとても手軽にしてくれました。 ブラウザ上で数分でモデルを学習できます。 でも、カメラの全画面を一括で見るため、 「何が写っているか」はわかっても「どこにあるか」はわかりません。

YOLOなどの物体検出フレームワークなら両方できますが、 環境構築、アノテーション、GPU学習など、 クリエイティブコーダーやインタラクションデザイナーにはハードルが高すぎます。

The gap

Easy classification, but no spatial awareness

Teachable Machine made image classification delightfully easy — train a model in minutes, right in your browser. But it looks at the entire camera frame at once. It can tell you what it sees, but not where.

YOLO and other object detection frameworks can do both — but setting them up involves environment configuration, annotation workflows, and GPU training pipelines. That's a steep jump for creative coders and interaction designers.

Teachable Machine単体 alone

  • 全画面の一括識別のみ
  • 空間的な情報がない
  • 物体の位置を区別できない
  • Whole-frame classification only
  • No spatial information
  • Can't distinguish location of objects

YOLO / 物体検出Object Detection

  • 環境構築が複雑
  • 手動でのバウンディングボックス付与
  • GPU学習が必要
  • Complex environment setup
  • Manual bounding-box annotation
  • GPU training required

Teachable Machine ROI

  • 学習済みのTeachable Machineモデルをそのまま使える
  • カメラ映像上にROI（関心領域）を自由に描画
  • 各ROIを独立して識別
  • 結果をOSCで送信し、あらゆるアプリと連携
  • Use your existing Teachable Machine model as-is
  • Draw square ROI regions on the camera feed
  • Each region classified independently
  • Results sent as OSC for live integration with any app
しくみ

枠を描いて、モデルを読み込むだけ。

カメラプレビュー上に正方形のROI領域を描画します。 各領域は個別に切り出され、Teachable Machineのモデルに入力されます。 識別結果はリアルタイムで表示され、OSCで送信されます。

How it works

Draw a box. Load a model. Done.

Draw one or more square ROI regions directly on the camera preview. Each region is cropped and fed into your Teachable Machine model independently. Classification results are displayed in real time and sent over OSC.

特徴

できること

Features

What you get

▢

複数のROI領域

必要なだけ領域を描画できます。各ROIが独立して識別を実行します。

Multiple ROI Regions

Draw as many regions as you need. Each one runs classification independently with its own results.

◎

正方形制約

Teachable Machineの224×224入力に合わせて、ROIは常に正方形。アスペクト比のゆがみを防ぎます。

Square Constraint

ROIs are always square to match Teachable Machine's 224×224 input, preventing aspect-ratio distortion.

⇝

OSC出力

識別結果をOSCで送信。Pure Data、Max/MSP、TouchDesigner、Unityなどと連携できます。

OSC Output

Classification results sent via OSC. Works with Pure Data, Max/MSP, TouchDesigner, Unity, and more.

☼

クラウド / ローカルモデル

Teachable MachineのURLを貼るか、ダウンロードした.zipを読み込み。オフラインでも動作します。

Cloud or Local Models

Paste a Teachable Machine URL or load a downloaded .zip. Works offline with local models.

✎

ROIリネーム

各ROIに名前をつけられます。OSCアドレスも自動で更新されます。

Renamable ROIs

Give each ROI a meaningful name. OSC addresses update automatically to match.

✓

選択的OSC送信

チェックボックスで送信するデータを選択。送信中はインジケーターで状態を確認できます。

Selective OSC Sending

Choose which classification results to send via checkboxes. Monitor active transmissions with a status indicator.

Use cases

What you can do with it

空間トリガー

「左のゾーンに物体Aがあり、かつ右のゾーンに物体Bがあったら、イベントCを実行」

移動検知

「物体Aが左のROIから右のROIに移動したら、音を鳴らす」

マルチゾーンインタラクション

テーブルやステージに複数のゾーンを設定し、各ゾーンに現れたものに個別に反応する。

ライブパフォーマンス

識別結果をOSCでPure DataやMax/MSPに送信し、サウンド・映像・照明をリアルタイム制御。

Spatial triggers

"If object A is in the left zone AND object B is in the right zone, trigger event C."

Movement detection

"When object A moves from the left ROI to the right ROI, play a sound."

Multi-zone interaction

Set up multiple zones on a table or stage and respond to what appears in each zone independently.

Live performance

Send classification results to Pure Data or Max/MSP via OSC to control sound, visuals, or lighting in real time.
